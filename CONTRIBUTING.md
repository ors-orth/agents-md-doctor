# Contributing

Thanks for considering a contribution.

## Development Status

This repository is in early setup. Please check the README and open issues before starting larger work.

## How to Contribute

1. Open an issue for bugs, feature ideas, or larger design changes.
2. Keep pull requests focused on one clear change.
3. Include tests or validation notes when behavior changes.
4. Update documentation when user-facing behavior changes.

## Commit Style

Use short, imperative commit messages, for example:

```text
Add AGENTS.md parser
Fix markdown heading detection
Document CLI usage
```
